# SOS Button App

Приложение для экстренных вызовов охраны с поддержкой push-уведомлений.

## Особенности

- Мгновенные SOS-вызовы с геолокацией
- Push-уведомления для охраны о новых вызовах
- Адаптивный интерфейс для мобильных устройств
- Звуковые и визуальные оповещения
- Возможность принятия/отклонения вызовов

## Установка и запуск

### Предварительные требования

- Node.js 16+ и npm
- MongoDB
- Браузер с поддержкой Push API (Chrome, Firefox, Edge)

### Шаги установки

1. Клонируйте репозиторий:
   ```
   git clone https://github.com/your-username/sos-button.git
   cd sos-button
   ```

2. Установите зависимости:
   ```
   npm install
   ```

3. Сгенерируйте VAPID ключи для push-уведомлений:
   ```
   npm run generate-vapid
   ```

4. Создайте файл `.env` в корне проекта и добавьте необходимые переменные окружения:
   ```
   MONGODB_URI=mongodb://localhost:27017/sos-app
   JWT_SECRET=your_jwt_secret_key
   PORT=5000
   ```

5. Запустите приложение в режиме разработки:
   ```
   npm run dev
   ```

## Настройка Push-уведомлений

### Для разработчиков

1. VAPID ключи генерируются автоматически при запуске `npm run generate-vapid` и сохраняются в файле `.env`.

2. Для тестирования push-уведомлений:
   - Зарегистрируйтесь как охранник
   - Дайте разрешение на отправку уведомлений в браузере
   - Создайте тестовый SOS-вызов с другого аккаунта

### Для пользователей

1. При первом входе в систему, браузер запросит разрешение на отправку уведомлений. Необходимо разрешить для получения оповещений о вызовах.

2. На устройствах Android рекомендуется добавить приложение на главный экран для лучшей работы push-уведомлений.

3. На устройствах iOS push-уведомления работают только в браузере Safari и только когда браузер открыт.

## Работа с push-уведомлениями

### Серверная часть

1. Отправка уведомлений происходит через сервис `notificationService.js` с помощью библиотеки `web-push`.

2. При новом SOS-вызове, сервер находит всех охранников и их подписки, затем отправляет push-уведомления.

### Клиентская часть

1. При авторизации пользователя с ролью "guard" (охранник), клиент запрашивает разрешение на push-уведомления и регистрирует Service Worker.

2. Service Worker обрабатывает входящие push-уведомления и показывает их пользователю, даже если браузер закрыт.

3. При клике на уведомление, пользователь перенаправляется на страницу с деталями вызова.

## Устранение неполадок

### Push-уведомления не работают

1. Проверьте, что браузер поддерживает Push API и Service Workers.
2. Убедитесь, что вы дали разрешение на отправку уведомлений.
3. Проверьте консоль браузера на наличие ошибок.
4. Убедитесь, что VAPID ключи сгенерированы и правильно настроены.

### Звук сирены не воспроизводится

1. Убедитесь, что файл `siren.mp3` доступен в директории `/public`.
2. Проверьте настройки звука в браузере и системе.
3. На мобильных устройствах звук может не воспроизводиться автоматически без взаимодействия пользователя.
